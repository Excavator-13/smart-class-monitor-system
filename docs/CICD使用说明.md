# Jenkins CI/CD 使用说明书

## 一、概述

本项目使用 Jenkins 实现持续集成与持续部署（CI/CD）。组员 git push 代码后，Jenkins 自动拉取代码并进行语法检查、构建验证。合并到 dev 分支时，自动将前端部署至云服务器。

- **Jenkins 地址**：http://39.106.209.208:49999
- **部署地址**：http://39.106.209.208

---

## 二、使用方法

### 2.1 查看构建结果

1. 浏览器打开 http://39.106.209.208:49999
2. 左侧分支列表找到你的分支，点击进入
3. 查看最新一条构建记录的状态

### 2.2 构建状态含义

| 状态 | 含义 | 操作 |
|---|---|---|
| ✅ 成功（Success） | 代码检查与构建均通过 | 无需操作 |
| ❌ 失败（Failure） | 存在语法错误或构建失败 | 点击构建记录 → Console Output → 查看错误详情 → 修复后重新 push |
| ⏭️ 跳过（Skipped） | 该阶段不适用于当前分支 | 无需操作 |

### 2.3 触发方式

- **自动触发**：git push 后，GitHub Webhook 通知 Jenkins 自动构建
- **手动触发**：在分支页面点击「立刻构建」（Build Now）

## 三、流水线说明

### 3.1 阶段说明

| 阶段 | 触发分支 | 说明 |
|---|---|---|
| Python 语法检查 | 所有分支 | 使用 flake8 检查 backend_ai/ 目录下 Python 代码规范 |
| 前端构建检查 | 所有分支 | 执行 npm install 和 vite build，验证前端能否正常构建 |
| 自动部署 | 仅 dev 分支 | 构建前端并部署到云服务器 /var/www/html/ 目录 |

### 3.2 自动部署内容

| 部署内容 | 源路径 | 目标路径 |
|---|---|---|
| 前端静态文件 | frontend/dist/ | /var/www/html/ |
| 后端 AI 代码 | backend_ai/ | /app/（暂不运行） |

### 3.3 未部署内容及原因

| 内容 | 原因 |
|---|---|
| Flask AI 推理服务 | 需要 GPU，在本地开发机运行 |
| SpringBoot 管理后端 | 开发阶段在本地运行 |
| GPU 相关依赖（torch/cuda） | 云服务器无 NVIDIA 显卡 |

## 四、配置文件

### 4.1 Jenkinsfile

项目根目录下的 Jenkinsfile 定义了完整的 CI/CD 流程。各阶段使用 `|| true` 或 `|| echo` 处理预期内的失败，避免阻塞流水线。

## 五、常见问题

**Q：为什么我 push 后 Jenkins 没反应？**

A：Webhook 触发有 10-30 秒延迟。如超过 1 分钟仍未触发，检查 GitHub Webhook 配置是否正常，或手动点击「立刻扫描多分支流水线」。

**Q：Python 检查报了 W391/E501 错误？**

A：W391（文件末尾空行）和 E501（行过长）为代码风格警告，不影响功能，但建议修复。

**Q：前端构建失败怎么办？**

A：确认 frontend/ 目录下 package.json 存在且依赖完整，本地先执行 `npm install && npm run build` 确认可通过后再 push。
