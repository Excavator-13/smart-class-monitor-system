## 5.2 人脸识别模块

### 5.2.1 人脸识别模块流程设计

图5-5 人脸识别模块流程图

```mermaid
flowchart TD
    A[前端采集人脸图像] --> B[Base64 编码]
    B --> C[POST /students/id/face]
    C --> D[Flask /face/feature/extract]
    D --> E[InsightFace 检测人脸]
    E --> F{检测到几张人脸?}
    F -->|0 张| G[返回错误 40002]
    F -->|多张| H[返回错误 40003]
    F -->|恰好 1 张| I[提取 512 维特征向量]
    I --> J[保存到 face_feature 表]
    J --> K[更新 student.face_registered = true]
    K --> L[返回注册成功]
    L --> M[后续视频帧到达]
    M --> N[InsightFace 检测帧中所有人脸]
    N --> O[与已注册特征计算余弦相似度]
    O --> P{最高相似度 >= 0.45?}
    P -->|是| Q[生成 face_recognized 事件]
    P -->|否| R[生成 stranger_detected 事件]
    Q --> S[绘制绿色标注框]
    R --> S
```

## 5.4 异常行为检测模块

### 5.4.1 异常行为检测模块流程设计

图5-13 异常行为检测模块流程图

```mermaid
flowchart TD
    A[视频帧到达] --> B[BehaviorService.detect_objects]
    B --> C[返回 objects 列表]
    C --> D[筛选 persons 和 phones]
    D --> E{phone_usage 规则启用?}
    E -->|是| F[遍历 person-phone 对]
    F --> G{phone 中心在人框内?}
    G -->|是| H[生成 phone_usage 事件]
    G -->|否| F
    E -->|否| I{head_down 规则启用?}
    H --> I
    I -->|是| J[遍历 persons]
    J --> K{head_down_ratio >= 阈值?}
    K -->|是| L[生成 head_down 事件]
    K -->|否| J
    I -->|否| M{crowd_gathering 启用?}
    L --> M
    M -->|是| N{人数 >= min_count?}
    N -->|是| O[计算人员中心点距离]
    O --> P{存在聚集?}
    P -->|是| Q[生成 crowd_gathering 事件]
    P -->|否| R[不触发]
    N -->|否| R
    M -->|否| R
    H --> S[EventService.observe]
    L --> S
    Q --> S
    S --> T{duration >= threshold?}
    T -->|是| U{cooldown 已过?}
    U -->|是| V[标记 confirmed]
    V --> W[AlertClient.push_alert]
    T -->|否| X[不确认]
    U -->|否| X
```
