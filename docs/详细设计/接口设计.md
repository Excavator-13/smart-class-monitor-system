# 接口设计

## 1. 接口概述

系统采用前后端分离架构，包含两个后端服务：
- **Spring Boot 后端**（端口 8080）：业务数据管理、用户认证、告警存储
- **Flask AI 后端**（端口 5000）：视频流分析、AI 推理、事件检测

前端通过 HTTP RESTful API 与后端交互，后端服务之间通过内部接口通信。

---

## 2. 通用规范

### 2.1 请求格式

- 内容类型：`application/json`
- 字符编码：UTF-8
- 认证方式：JWT Token（前端请求）/ X-Internal-Token（内部服务调用）

### 2.2 响应格式

**标准响应结构**：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

**分页响应结构**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [...],
    "page": 1,
    "page_size": 10,
    "total": 100
  }
}
```

**错误响应结构**：

```json
{
  "code": 40001,
  "message": "invalid image",
  "data": null
}
```

### 2.3 错误码定义

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 40001 | 无效图像 |
| 40002 | 未检测到人脸 |
| 40003 | 检测到多张人脸 |
| 40401 | 视频流不存在 |

## 4. Flask AI 后端接口

### 4.1 视频流接口

#### GET /video_feed/{stream_id}

**说明**：返回 MJPEG 实时流

**路径参数**：`stream_id` - 视频源 ID

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|

**响应**：`multipart/x-mixed-replace; boundary=frame`

每帧格式：

```
--frame
Content-Type: image/jpeg

<JPEG 二进制数据>
```

## 6. 数据库表结构

### 6.1 用户表

```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(64) NOT NULL UNIQUE,
    password_hash VARCHAR(256) NOT NULL,
    nickname VARCHAR(128),
    role VARCHAR(16) DEFAULT 'teacher',
    status VARCHAR(16) DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 学生表

```sql
CREATE TABLE student (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    student_no VARCHAR(64) NOT NULL UNIQUE,
    name VARCHAR(128) NOT NULL,
    class_name VARCHAR(64),
    face_registered TINYINT DEFAULT 0,
    status VARCHAR(16) DEFAULT 'active',
    last_seen DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.3 人脸特征表

```sql
CREATE TABLE face_feature (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    student_id BIGINT NOT NULL,
    feature_vector TEXT,
    image_path VARCHAR(512),
    quality_score DOUBLE,
    enabled TINYINT DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (student_id) REFERENCES student(id)
);
```

### 6.4 视频流表

```sql
CREATE TABLE video_stream (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    stream_id VARCHAR(64) NOT NULL UNIQUE,
    stream_name VARCHAR(128),
    rtmp_url VARCHAR(512),
    status VARCHAR(16) DEFAULT 'enabled',
    remark VARCHAR(256),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.5 危险区域表

```sql
CREATE TABLE danger_zone (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    stream_id VARCHAR(64) NOT NULL,
    zone_name VARCHAR(128),
    zone_type VARCHAR(32),
    coordinates TEXT,
    threshold_seconds INT DEFAULT 2,
    enabled TINYINT DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (stream_id) REFERENCES video_stream(stream_id)
);
```

### 6.6 行为规则表

```sql
CREATE TABLE behavior_rule (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    rule_type VARCHAR(64) NOT NULL UNIQUE,
    rule_name VARCHAR(128),
    threshold_seconds INT DEFAULT 3,
    confidence_threshold DOUBLE DEFAULT 0.6,
    cooldown_seconds INT DEFAULT 45,
    enabled TINYINT DEFAULT 1,
    config_json TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.7 告警事件表

```sql
CREATE TABLE alert_event (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    stream_id VARCHAR(64),
    student_id BIGINT,
    alert_type VARCHAR(64) NOT NULL,
    alert_name VARCHAR(128),
    level VARCHAR(16) DEFAULT 'warning',
    confidence DOUBLE,
    status VARCHAR(16) DEFAULT 'unhandled',
    target_info TEXT,
    zone_id BIGINT,
    snapshot_path VARCHAR(512),
    record_path VARCHAR(512),
    occurred_at DATETIME,
    handled_at DATETIME,
    remark VARCHAR(512),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (stream_id) REFERENCES video_stream(stream_id),
    FOREIGN KEY (student_id) REFERENCES student(id)
);
```

### 6.8 操作日志表

```sql
CREATE TABLE operation_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT,
    action VARCHAR(128),
    target_type VARCHAR(64),
    target_id BIGINT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES user(id)
);
```
