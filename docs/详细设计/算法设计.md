# 算法设计

## 1. 人脸检测与特征提取算法

### 1.1 算法概述

系统采用 InsightFace 框架的 ArcFace 模型进行人脸检测和特征提取。InsightFace 是一个开源的深度学习人脸分析工具包，集成了人脸检测、对齐和特征提取功能。

### 1.2 模型参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 模型名称 | buffalo_l | InsightFace 预训练模型包 |
| 特征维度 | 512 | 每个人脸输出 512 维归一化向量 |
| 检测尺寸 | 640×640 | 输入图像缩放尺寸 |
| 相似度阈值 | 0.45 | 余弦相似度判定阈值 |

### 1.3 推理后端选择

系统根据运行环境自动选择最优 ONNX 推理提供者：

```python
def _detect_onnx_providers(device):
    if device == "cpu":
        return ["CPUExecutionProvider"], -1
    is_apple_silicon = platform.system() == "Darwin" and platform.machine() == "arm64"
    if device == "cuda" or (device is None and not is_apple_silicon):
        return ["CUDAExecutionProvider", "CPUExecutionProvider"], 0
    if is_apple_silicon:
        return ["CoreMLExecutionProvider", "CPUExecutionProvider"], -1
    return ["CPUExecutionProvider"], -1
```

优先级：CUDA (GPU) > CoreML (Apple Silicon) > CPU

### 1.4 降级方案

当 InsightFace 不可用时，系统降级为 OpenCV Haar 级联检测器：

```python
def _faces(self, frame):
    if self.model is not None and hasattr(self.model, "get"):
        return list(self.model.get(frame))
    # 降级方案
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    detector = cv2.CascadeClassifier(cv2.data.haarcascades + "haarcascade_frontalface_default.xml")
    boxes = detector.detectMultiScale(gray, 1.1, 4)
    faces = []
    for x, y, w, h in boxes:
        roi = cv2.resize(gray[y:y+h, x:x+w], (32, 16)).astype("float32").flatten()
        feature = np.zeros(self.feature_dim, dtype="float32")
        feature[:min(len(roi), self.feature_dim)] = roi[:self.feature_dim]
        norm = np.linalg.norm(feature) or 1.0
        faces.append({"bbox": [int(x), int(y), int(x+w), int(y+h)], "embedding": feature / norm})
    return faces
```

降级方案仅用于开发调试，识别精度远低于 InsightFace。

## 2. 人脸识别比对算法

### 2.1 算法概述

人脸识别采用余弦相似度比对。对于当前帧中每张人脸的 embedding 向量，与缓存中所有已注册学生的 feature_vector 进行比对，取最高相似度。

### 2.2 余弦相似度计算

```python
similarity = dot(a, b) / (||a|| × ||b||)
```

其中：
- `a` 为当前帧人脸的 512 维 embedding 向量
- `b` 为已注册学生的 512 维 feature_vector
- `dot(a, b)` 为向量点积
- `||a||` 和 `||b||` 为向量的 L2 范数

### 2.3 比对流程

```python
def recognize_frame(self, frame, feature_cache):
    faces = self._faces(frame)
    events = []
    for idx, face in enumerate(faces):
        embedding = np.asarray(face["embedding"], dtype=float)
        bbox = face["bbox"]
        best = (-1.0, None)
        
        # 遍历所有已注册学生
        for student in feature_cache.values():
            vector = np.asarray(parse_json_field(student.get("feature_vector"), []), dtype=float)
            if vector.size != embedding.size or vector.size == 0:
                continue
            # 计算余弦相似度
            similarity = float(np.dot(embedding, vector) / 
                             ((np.linalg.norm(embedding) * np.linalg.norm(vector)) or 1.0))
            if similarity > best[0]:
                best = (similarity, student)
        
        # 判定结果
        if best[1] and best[0] >= self.similarity_threshold:
            # 识别成功
            events.append({
                "event_type": "face_recognized",
                "confidence": best[0],
                "level": "info",
                "target": {
                    "track_id": f"face_{idx + 1}",
                    "student_id": best[1].get("student_id"),
                    "student_name": best[1].get("student_name"),
                    "bbox": bbox
                }
            })
        else:
            # 陌生人
            events.append({
                "event_type": "stranger_detected",
                "confidence": max(best[0], 0.0),
                "level": "warning",
                "target": {"track_id": f"face_{idx + 1}", "bbox": bbox}
            })
    return events
```

### 2.4 性能优化

- 特征缓存：ConfigClient 缓存所有已注册学生的特征向量，避免每帧查询数据库
- 增量刷新：新增人脸注册后，通过 `/face/features/reload` 接口增量更新缓存
- 向量归一化：注册时特征向量已归一化，比对时可简化计算

def detect_objects(self, frame):

## 4. 区域入侵检测算法

### 4.1 算法概述

区域入侵检测基于多边形几何判定。系统支持三种事件类型：
- 入侵（intrusion）：人员脚部在危险区域内
- 停留（stay）：人员在危险区域内持续超过阈值时间
- 接近（approach）：人员脚部在危险区域外但距离小于安全距离

### 4.2 脚部坐标计算

人员目标的脚部坐标取 bbox 底边中点：

```python
def bbox_foot_point(bbox):
    """bbox = [x1, y1, x2, y2]"""
    return ((bbox[0] + bbox[2]) / 2, bbox[3])
```

使用脚部坐标而非 bbox 中心，更符合人员实际位置。

### 4.3 点在多边形内判定（射线法）

```python
def point_in_polygon(point, polygon):
    """
    point: (x, y)
    polygon: [(x1, y1), (x2, y2), ...]
    """
    x, y = point
    n = len(polygon)
    inside = False
    p1x, p1y = polygon[0]
    for i in range(n + 1):
        p2x, p2y = polygon[i % n]
        if y > min(p1y, p2y):
            if y <= max(p1y, p2y):
                if x <= max(p1x, p2x):
                    if p1y != p2y:
                        xinters = (y - p1y) * (p2x - p1x) / (p2y - p1y) + p1x
                    if p1x == p2x or x <= xinters:
                        inside = not inside
        p1x, p1y = p2x, p2y
    return inside
```

算法原理：从待判定点向右发射水平射线，统计与多边形边的交点数量。交点为奇数则在多边形内，偶数则在多边形外。

### 4.4 点到多边形距离计算

```python
def distance_point_to_polygon(point, polygon):
    """计算点到多边形每条边的最短距离，取最小值"""
    x, y = point
    n = len(polygon)
    min_dist = float('inf')
    for i in range(n):
        p1 = polygon[i]
        p2 = polygon[(i + 1) % n]
        dist = distance_point_to_segment(point, p1, p2)
        min_dist = min(min_dist, dist)
    return min_dist

def distance_point_to_segment(point, p1, p2):
    """计算点到线段的距离"""
    x, y = point
    x1, y1 = p1
    x2, y2 = p2
    # 线段长度的平方
    l2 = (x2 - x1) ** 2 + (y2 - y1) ** 2
    if l2 == 0:
        return ((x - x1) ** 2 + (y - y1) ** 2) ** 0.5
    # 投影参数 t
    t = max(0, min(1, ((x - x1) * (x2 - x1) + (y - y1) * (y2 - y1)) / l2))
    # 投影点
    proj_x = x1 + t * (x2 - x1)
    proj_y = y1 + t * (y2 - y1)
    return ((x - proj_x) ** 2 + (y - proj_y) ** 2) ** 0.5
```

### 4.5 坐标归一化

区域坐标支持两种格式：
- 归一化坐标（0-1 范围）：与图像尺寸无关，推荐
- 像素坐标：与图像尺寸相关

解析时自动判断：

```python
def parse_polygon_coordinates(coords):
    """解析坐标，支持归一化和像素坐标"""
    polygon = []
    for x, y in coords:
        if 0 <= x <= 1 and 0 <= y <= 1:
            # 归一化坐标，后续根据图像尺寸转换
            polygon.append((x, y))
        else:
            # 像素坐标
            polygon.append((x, y))
    return polygon
```

### 4.6 接近预警置信度

```python
confidence = max(0.0, 1.0 - distance / max(safe_distance, 1e-9))
```

距离越近置信度越高，距离等于 safe_distance 时置信度为 0。

## 5. 异常行为检测算法

### 5.1 违规使用手机检测

**判定条件**：
1. phone 目标的置信度 >= confidence_threshold（默认 0.6）
2. phone bbox 的中心点落在 person bbox 内部

```python
def _iou_like_relation(person_bbox, object_bbox):
    """判断 object 中心是否在 person 框内"""
    px1, py1, px2, py2 = person_bbox
    ox1, oy1, ox2, oy2 = object_bbox
    cx = (ox1 + ox2) / 2
    cy = (oy1 + oy2) / 2
    return px1 <= cx <= px2 and py1 <= cy <= py2
```

**检测流程**：

```python
for person in persons:
    for phone in phones:
        if phone.get("confidence", 0) >= phone_threshold and \
           _iou_like_relation(person["bbox"], phone["bbox"]):
            detections.append({
                "event_type": "phone_usage",
                "confidence": min(1.0, float(phone.get("confidence", 0))),
                "level": "warning",
                "target": {"track_id": person.get("track_id"), "bbox": person["bbox"]},
                "threshold_seconds": float(phone_rule.get("threshold_seconds", 3)),
                "cooldown_seconds": float(phone_rule.get("cooldown_seconds", 45))
            })
```

### 5.2 长时间低头检测

**判定条件**：
- head_down_ratio >= confidence_threshold（默认 0.6）

head_down_ratio 由 YOLO 姿态估计或自定义分类器输出，表示低头的置信度。

```python
for person in persons:

        detections.append({
            "event_type": "head_down",
            "confidence": float(ratio),
            "level": "warning",
            "target": {"track_id": person.get("track_id"), "bbox": person["bbox"]},
            "threshold_seconds": float(head_rule.get("threshold_seconds", 3)),
            "cooldown_seconds": float(head_rule.get("cooldown_seconds", 45))
        })
```

### 5.3 异常聚集检测

**判定条件**：
1. 画面中人员数量 >= min_count（默认 4 人）
2. 存在 min_count 人聚集在 max_center_distance 范围内（默认 0.15 归一化距离）

**聚集判定算法**：

```python
def _has_crowd(self, persons, min_count, max_distance):
    centers = [_bbox_center(person["bbox"]) for person in persons]
    # 枚举所有 min_count 人组合
    for group in combinations(centers, min_count):
        distances = []
        for a, b in combinations(group, 2):
            distances.append(((a[0] - b[0]) ** 2 + (a[1] - b[1]) ** 2) ** 0.5)
        # 组合内所有距离均 <= max_distance
        if distances and max(distances) <= max_distance:
            return True
    return False

def _bbox_center(bbox):
    return ((bbox[0] + bbox[2]) / 2, (bbox[1] + bbox[3]) / 2)
```

算法复杂度：O(C(n, k) × k²)，其中 n 为人员数量，k 为 min_count。

### 5.4 明火检测

**检测参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| confidence_threshold | 0.25 | 置信度阈值 |
| min_bbox_area | 1000 | 最小检测框面积（像素） |
| max_detections | 20 | 单帧最大检测数 |

**检测流程**：

```python
def detect(self, stream_id, frame, rules=None):
    if self.model is None:
        return []
    rule = (rules or {}).get("flame_detected", {})
    threshold = float(rule.get("confidence_threshold", self.confidence_threshold))
    
    results = self.model(frame, verbose=False)
    detections = []
    
    for r in results:
        boxes = getattr(r, "boxes", None)
        if boxes is None:
            continue
        for box in boxes:
            conf = float(box.conf[0])
            if conf < threshold:
                continue
            x1, y1, x2, y2 = [float(v) for v in box.xyxy[0]]
            area = (x2 - x1) * (y2 - y1)
            if area < self.min_bbox_area:
                continue
            
            level = self._classify_level(conf)
            detections.append({
                "event_type": "flame_detected",
                "confidence": round(conf, 4),
                "level": level,
                "target": {"track_id": f"fire_{idx}", "bbox": [x1, y1, x2, y2]},
                "threshold_seconds": float(rule.get("threshold_seconds", 0)),
                "cooldown_seconds": float(rule.get("cooldown_seconds", 10))
            })
    
    # 按置信度降序排列，截取前 max_detections 个
    detections = sorted(detections, key=lambda d: d["confidence"], reverse=True)
    detections = detections[:self.max_detections]
    return detections

def _classify_level(self, confidence):
    if confidence >= 0.8:
        return "high"
    elif confidence >= 0.6:
        return "warning"
    return "info"
```

## 6. 事件确认机制

### 6.1 算法概述

EventService 实现事件确认机制，避免瞬时检测误触发告警。每个事件由 (stream_id, event_type, track_key) 三元组唯一标识，需持续满足条件超过 threshold_seconds 才会确认并推送告警。

### 6.2 事件状态管理

```python
@dataclass
class EventState:
    first_seen: float      # 首次检测到时间
    last_seen: float       # 最后检测到时间
    last_alert_at: float   # 上次告警推送时间
    event_id: str          # 事件 ID
    event_status: str      # 事件状态 (candidate/confirmed)
```

### 6.3 确认算法

```python
def observe(self, stream_id, event_type, track_key, confidence, 
            threshold_seconds=0.0, cooldown_seconds=None, level="warning", 
            target=None, zone=None, now=None):
    current = time.time() if now is None else now
    key = (stream_id, event_type, track_key)
    
    # 获取或创建事件状态
    state = self._states.get(key)
    if state is None:
        state = EventState(first_seen=current, last_seen=current)
        self._states[key] = state
    state.last_seen = current
    
    # 计算持续时间
    duration = current - state.first_seen
    cooldown = self.default_cooldown_seconds if cooldown_seconds is None else cooldown_seconds
    
    # 判定是否确认
    should_confirm = (
        duration >= threshold_seconds and
        current - state.last_alert_at >= cooldown and
        state.event_status != "confirmed"
    )
    
    if should_confirm:
        state.event_status = "confirmed"
        state.last_alert_at = current
    
    # 构建事件
    event = self.build_event(
        stream_id=stream_id,
        event_type=event_type,
        confidence=confidence,
        level=level,
        target=target,
        zone=zone,
        duration_seconds=duration,
        status=state.event_status,
        event_id=state.event_id
    )
    self.add_event(event)
    return event, should_confirm
```

### 6.4 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| threshold_seconds | 3 | 持续时间阈值（秒） |
| cooldown_seconds | 45 | 告警冷却时间（秒） |
| max_items | 500 | 事件队列最大长度 |

### 6.5 防重复推送机制

- cooldown_seconds：同一事件在冷却期间内不会重复推送
- event_status：已确认事件不会重复确认
- event_id：同一事件的多次检测共享相同的 event_id

## 7. 断流检测与自动重连算法

### 7.1 断流检测

StreamManager 在拉流循环中持续读取帧，每次成功读取后更新 `last_frame_at` 时间戳。`get_frame()` 方法在返回帧之前检查：

```python
def get_frame(self, stream_id):
    state = self.ensure_stream(stream_id) or self._states.get(stream_id)
    if not state:
        return None
    if state.last_frame_at and time.time() - state.last_frame_at > self.offline_after_seconds:
        state.online = False
    return None if state.frame is None else state.frame.copy()
```

默认 `offline_after_seconds = 10`，超过 10 秒无新帧则标记为离线。

### 7.2 自动重连

拉流线程在 VideoCapture 读取失败或连接失败时，释放当前 capture 对象，等待 `reconnect_interval_seconds`（默认 3 秒）后重新创建 VideoCapture 连接：

```python
def _read_loop(self, stream_id):
    while not self._stop.is_set():
        state = self._states[stream_id]
        if not state.rtmp_url:
            state.online = False
            time.sleep(self.reconnect_interval_seconds)
            continue
        
        cap = self._captures.get(stream_id)
        if cap is None or not cap.isOpened():
            cap = cv2.VideoCapture(state.rtmp_url)
            self._captures[stream_id] = cap
            if not cap.isOpened():
                state.online = False
                time.sleep(self.reconnect_interval_seconds)
                continue
        
        ok, frame = cap.read()
        if not ok or frame is None:
            state.online = False
            cap.release()
            time.sleep(self.reconnect_interval_seconds)
            continue
        
        with self._lock:
            state.frame = frame
            state.online = True
            state.last_frame_at = time.time()
```

### 7.3 FPS 计算

使用滑动窗口法，以 1 秒为窗口周期：

```python
frame_count = 0
fps_window_start = time.time()

while not self._stop.is_set():
    # ... 读取帧 ...
    frame_count += 1
    now = time.time()
    elapsed = now - fps_window_start
    if elapsed >= 1.0:
        state.fps = frame_count / elapsed
        frame_count = 0
        fps_window_start = now
```

## 8. 风险指数计算算法

### 8.1 算法概述

风险指数根据当前视频流的事件类型和数量计算综合分数（0-100），用于展示当前风险等级。

### 8.2 事件权重

| 事件类型 | 权重 | 说明 |
|----------|------|------|
| flame_detected | 30 | 明火检测，最高优先级 |
| danger_zone_intrusion | 15 | 危险区域入侵 |
| stranger_detected | 10 | 陌生人出现 |
| phone_usage | 8 | 违规使用手机 |
| head_down | 5 | 长时间低头 |
| crowd_gathering | 10 | 异常聚集 |

### 8.3 计算公式

```python
def calculate_risk_score(events):
    high_count = sum(1 for e in events if e.get("level") == "high")
    warning_count = sum(1 for e in events if e.get("level") == "warning")
    
    # 风险分数 = high 事件数 × 30 + warning 事件数 × 10
    risk_score = min(100, high_count * 30 + warning_count * 10)
    
    # 风险等级
    if high_count > 0:
        risk_level = "high"
    elif warning_count > 3:
        risk_level = "warning"
    else:
        risk_level = "low"
    
    return risk_score, risk_level
```

### 8.4 风险等级

| 等级 | 分数范围 | 说明 |
|------|----------|------|
| low | 0-30 | 低风险，持续监控 |
| warning | 31-70 | 中等风险，需关注 |
| high | 71-100 | 高风险，需立即处理 |
