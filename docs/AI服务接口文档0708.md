# 智慧教室实时行为分析与安全监测系统
# AI 服务接口文档

版本：V0.2  
日期：2026-07-08  
服务：Flask AI 分析服务  
默认地址：`http://39.106.209.208:5000`

说明：AI 服务整体使用 Flask 框架统一封装接口；目标检测使用 YOLO，人脸识别使用 **OpenCV + InsightFace**（ArcFace 模型，输出 **512 维**特征向量）。

---

## 1. 接口约定

### 1.1 Base URL

前端通过环境变量访问 AI 服务：

```env
VITE_AI_BASE=http://39.106.209.208:5000
```

SpringBoot 后端通过配置项访问 AI 服务：

```yaml
ai:
  base-url: http://39.106.209.208:5000
```

### 1.2 统一 JSON 返回格式

除 MJPEG 视频流接口外，其他接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": "2026-07-07T10:20:30+08:00",
  "trace_id": "req_20260707102030001"
}
```

错误示例：

```json
{
  "code": 40001,
  "message": "stream_id is required",
  "data": null,
  "timestamp": "2026-07-07T10:20:30+08:00",
  "trace_id": "req_20260707102030001"
}
```

### 1.3 AI 接口路径约定

为与当前前端接口文档保持一致，Flask AI 服务接口不额外添加 `/ai` 前缀，直接使用以下路径：

- 实时视频流：`/video_feed/{stream_id}`
- 实时事件：`/analysis/events`
- 模型状态：`/model/status`
- 人脸特征提取：`/face/feature/extract`
- 配置刷新：`/config/reload`

### 1.4 通用错误码

| code | 含义 |
|---:|---|
| 0 | 成功 |
| 40001 | 参数缺失或格式错误 |
| 40002 | 图片中未检测到人脸 |
| 40003 | 图片中检测到多个人脸 |
| 40401 | 视频源不存在 |
| 50001 | AI 模型未加载 |
| 50002 | 视频流读取失败 |
| 50003 | SpringBoot 告警入库失败 |
| 50004 | 截图保存失败 |

## 2. 前端直接调用的 AI 接口

## 2.1 获取处理后实时视频流

```http
GET /video_feed/{stream_id}
```

### 使用场景

单路实时监控页面播放 AI 标注后的视频画面。前端直接用 `<img>` 渲染：

```html
<img :src="`${VITE_AI_BASE}/video_feed/${streamId}`" />
```

### Path 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `stream_id` | string | 是 | 视频源 ID，例如 `classroom_01` |

### Query 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `annotate` | boolean | 否 | true | 是否返回带标注画面 |
| `modules` | string | 否 | all | 启用模块，逗号分隔，如 `face,zone,behavior,fire` |

### 返回

`Content-Type: multipart/x-mixed-replace; boundary=frame`

返回 MJPEG 视频流，不使用统一 JSON 格式。

### 错误场景处理

| 场景 | HTTP 状态码 | 响应说明 | 前端处理建议 |
|---|---|---|---|
| `stream_id` 不存在 | 404 | `{"code":40401,"message":"stream not found"}` | 显示"视频源不存在"占位图 |
| 视频源未推流（拉流失败） | 503 | 返回最后一帧覆盖错误文字，或返回占位 JPEG | 显示"视频源离线"状态，保留重连按钮 |
| 拉流中断（进行中） | MJPEG 流终止 | 连接关闭，`<img>` 触发 `@error` 事件 | 前端 `@error` 回调中显示"连接断开，重连中…"，每 3s 自动重试 |
| 模型未加载 | 200 MJPEG | 返回原始无标注画面 | 正常运行，页面提示"AI 标注不可用" |

### 前端联调备注

- 该接口用于实时画面，不用于告警列表。
- 若视频流断开，前端应显示"视频源异常/重连中"。
- 前端不需要自己画框，标注由 AI 服务完成。

## 2.2 查询实时 AI 候选事件

```http
GET /analysis/events
```

### 使用场景

前端右侧实时告警卡片、视频底部实时研判条、AI 事件时间线。

### Query 参数

| 参数 | 类型 | 必填 | 示例 | 说明 |
|---|---|---|---|---|
| `stream_id` | string | 否 | `classroom_01` | 视频源 ID |
| `event_type` | string | 否 | `phone_usage` | 事件类型 |
| `level` | string | 否 | `warning` | `info`、`warning`、`high` |
| `limit` | int | 否 | 20 | 返回最近 N 条 |
| `since` | string | 否 | `2026-07-07T10:00:00+08:00` | 查询某时间之后事件 |

### 返回示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "event_id": "evt_20260707102030001",
        "stream_id": "classroom_01",
        "event_type": "phone_usage",
        "event_name": "使用手机",
        "level": "warning",
        "event_status": "candidate",
        "confidence": 0.86,
        "occurred_at": "2026-07-07T10:20:30+08:00",
        "duration_seconds": 3.2,
        "target": {
          "track_id": "person_3",
          "student_id": "2024001",
          "student_name": "张三",
          "bbox": [120, 80, 260, 420]
        },
        "zone": null,
        "snapshot_path": null
      }
    ]
  },
  "timestamp": "2026-07-07T10:20:31+08:00",
  "trace_id": "req_20260707102031001"
}
```

### 说明

- 该接口返回 AI 服务内存中的"候选事件"，不一定已经入库。
- 正式告警列表仍由 SpringBoot `/alerts` 提供。
- **初版建议**: 前端每 3-5 秒轮询一次。
- **后续扩展**: 可升级为 WebSocket 推送，路径为：AI 确认告警 → SpringBoot `/alerts/ai` 入库 → SpringBoot 通过 WebSocket/SSE 推送给前端。**注意**：AI 不直接通过 WebSocket 推送前端，避免绕过 SpringBoot 导致告警未入库即展示、状态不一致的问题。

## 5. 事件类型与字段枚举

### 5.1 event_type / alert_type

| 值 | 中文名称 | 来源模块 | 状态 |
|---|---|---|---|---|
| `face_recognized` | 已识别人员 | 人脸识别 | ✅ 已设计 |
| `stranger_detected` | 陌生人出现 | 人脸识别 | ✅ 已设计 |
| `danger_zone_intrusion` | 危险区域入侵 | 危险区域检测 | ✅ 已设计 |
| `danger_zone_stay` | 危险区域停留超时 | 危险区域检测 | ✅ 已设计 |
| `danger_zone_approach` | 危险区域接近预警 | 危险区域检测 | ✅ 新增 |
| `phone_usage` | 使用手机 | 目标行为检测 | ✅ 已设计 |
| `head_down` | 长时间低头 | 行为检测扩展 | ✅ 已设计 |
| `leave_seat` | 长时间离座 | 行为检测扩展 | ✅ 已设计 |
| `flame_detected` | 明火检测 | 明火检测 | ✅ 已设计 |
| `fall_detected` | 人员摔倒 | 行为检测扩展 | 🔵 后续扩展 |
| `stream_offline` | 视频流中断 | 视频流管理 | ✅ 已设计 |

### 5.1.1 `danger_zone_approach` 接近预警检测逻辑

任务清单明确要求"人员距离该区域边缘低于安全距离时进行告警"。此事件在人员尚未进入危险区域、但脚点到区域 polygon 边缘的最短距离小于配置的 `safe_distance` 阈值时触发，属于**提前预警**，比 `danger_zone_intrusion` 更早触发。

| 检测条件 | 说明 |
|---|---|
| 人员脚点未在 polygon 内 | 排除已进入区域的情况（由 `intrusion` 处理） |
| 脚点到 polygon 边缘最短距离 < `safe_distance` | 距离阈值从规则 `config_json.safe_distance` 读取，默认归一化坐标 0.05 |

> **注意**: `safe_distance` 参数需通过 SpringBoot `/rules` 接口的 `config_json` 字段下发。

### 5.2 level

| 值 | 含义 |
|---|---|
| `info` | 普通识别信息 |
| `warning` | 一般异常，需要关注 |
| `high` | 高风险，需要优先处理 |

### 5.3 alert status

正式告警状态由 SpringBoot 管理，建议枚举：

| 值 | 含义 |
|---|---|
| `unhandled` | 未处理 |
| `processing` | 处理中 |
| `handled` | 已处理 |
| `false_alarm` | 误报 |
| `ignored` | 忽略 |

### 5.4 候选事件状态 vs 正式告警状态

AI 服务内存中的"候选事件"与 SpringBoot 管理的"正式告警"是两套独立体系，字段名需区分：

| 层级 | 字段名 | 说明 | 取值 |
|---|---|---|---|
| 候选事件（AI 内存） | `event_status` | 候选事件的审核状态 | `candidate`（待确认）、`confirmed`（已确认入库）、`expired`（超时丢弃） |
| 正式告警（SpringBoot DB） | `alert_status` | 正式告警的处理状态 | `unhandled`、`processing`、`handled`、`false_alarm`、`ignored` |

> **注意**: 原候选事件结构中 `"status": "candidate"` 已改为 `"event_status": "candidate"`，与 SpringBoot 告警的 `alert_status` 明确区分，避免混淆。

### 5.5 `snapshot_path` 与 `snapshot_url` 字段映射

| 字段名 | 使用方向 | 含义 |
|---|---|---|
| `snapshot_path` | AI → SpringBoot（入库） | AI 服务器上的截图文件路径，如 `/snapshots/20260707/evt_001.jpg` |
| `snapshot_url` | SpringBoot → 前端（查询） | 可供前端访问的截图 HTTP URL，如 `/static/snapshots/20260707/evt_001.jpg` |

> **映射规则**: SpringBoot 入库时将 `snapshot_path` 转换为对外可访问的 `snapshot_url`，两个字段名语义合理，不再统一。AI 文档中使用 `snapshot_path`，前端文档中使用 `snapshot_url`。

### 5.6 候选事件 → 告警入库字段映射表

AI 服务将候选事件（§2.2 返回结构）确认为正式告警后，调用 SpringBoot `/alerts/ai` 入库时，字段映射如下：

| AI 候选事件字段 | SpringBoot 告警入库字段 | 转换规则 |
|---|---|---|
| `event_id` | `event_id` | 直接透传 |
| `stream_id` | `stream_id` | 直接透传 |
| `event_type` | `alert_type` | 直接映射（值相同） |
| `event_name` | `alert_name` | 直接映射 |
| `level` | `level` | 直接透传 |
| `confidence` | `confidence` | 直接透传 |
| `occurred_at` | `occurred_at` | 直接透传 |
| `duration_seconds` | `duration_seconds` | 直接透传 |
| `target.student_id` | `student_id` | 提升为顶级字段 |
| `target.track_id`, `target.bbox` | `target_info.track_id`, `target_info.bbox` | 嵌套→扁平（去掉 `student_id` 和 `student_name`） |
| `target.student_name` | — | 不入库（SpringBoot 可自行关联查询） |
| `zone.zone_id` | `zone_id` | 提升为顶级字段 |
| `zone.zone_name` | — | 不入库（SpringBoot 可自行关联查询） |

## 7. 联调顺序建议

1. 前端配置 `VITE_AI_BASE`，确认 `/model/status` 可访问。
2. 配置 Nginx RTMP，确认 OBS/手机可推流。
3. Flask 实现 `/video_feed/{stream_id}`，前端可看到实时画面。
4. Flask 返回 mock `/analysis/events`，前端完成实时事件展示。
5. SpringBoot 实现 `/students/{id}/face` 调 Flask `/face/feature/extract`。
6. SpringBoot 实现 `/zones`、`/rules`，Flask 拉取配置。
7. Flask 触发正式告警，调用 SpringBoot `/alerts/ai` 入库。
8. 前端通过 SpringBoot `/alerts` 查看告警记录和截图。

## 10. 配置刷新机制

AI 服务的配置（人脸特征、区域、规则、视频源）通过以下两种模式保持与 SpringBoot 同步：

### 10.1 推模式（Push）

SpringBoot 侧发生变更时，主动调用 AI 服务刷新接口：

| 变更类型 | SpringBoot 调用接口 | 说明 |
|---|---|---|
| 人脸注册/更新/删除 | `POST /face/features/reload` | AI 重新拉取人脸特征库 |
| 区域/规则/视频源变更 | `POST /config/reload` | 可指定 `stream_id` 和 `reload_items` 精确刷新 |

### 10.2 拉模式（Pull）

AI 服务启动后，定时从 SpringBoot 轮询拉取配置作为兜底：

| 配置项 | 轮询接口 | 默认间隔 | 说明 |
|---|---|---|---|
| 视频源列表 | `GET {SPRING_API_BASE}/streams` | 60s | 检测新增/删除视频源 |
| 区域配置 | `GET {SPRING_API_BASE}/zones?stream_id=` | 120s | 检测区域变更 |
| 行为规则 | `GET {SPRING_API_BASE}/rules` | 120s | 检测规则变更 |
| 人脸特征库 | `GET {SPRING_API_BASE}/students/face-features` | 300s | 兜底同步 |

### 10.3 优先级与降级

- **优先级**: 推模式 > 拉模式。收到 Push 通知后立即刷新对应配置项，并重置该配置项的轮询计时器。
- **降级**: 若 Push 通知失败（如 SpringBoot 不可达），AI 依赖拉模式兜底，下一个轮询周期自动同步。
