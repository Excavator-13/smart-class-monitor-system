整个系统**四大模块的完整依赖调用流程**，每个模块分“调用谁（数据从哪来/主动找谁）”和“被谁调用（给谁提供什么）”两部分写清楚：

### 1. 前端模块

- **调用谁（主动请求）**：
  - **AI 算法**：通过 `<img>` 标签请求 Flask 获取 MJPEG 实时视频流（带画框）；请求 Flask 获取实时候选事件。
  - **SpringBoot 管理端**：请求登录鉴权、人员增删改查、区域/规则配置、告警列表查询、视频源状态查询等所有结构化业务接口。
  - **云服务器**：根据 SpringBoot 返回的文件路径，直接向 Nginx 请求静态截图或录像文件。
- **被谁调用（提供给谁）**：
  - 无。前端是整个系统的最终展示层，只出不进。

### 2. AI 算法分析模块

- **调用谁（主动请求）**：
  - **云服务器**：启动时通过 OpenCV (`cv2.VideoCapture`) 拉取 Nginx 的 RTMP 视频流。
  - **SpringBoot 管理端**：检测到异常行为时，主动调 SpringBoot 接口将告警事件入库（让 SpringBoot 统一管理告警数据）。
- **被谁调用（提供给谁）**：
  - **前端**：向前端提供 `/video_feed` 接口，返回画好框的 MJPEG 流；提供 `/analysis/events` 接口，返回纯文本的实时事件供前端展示。
  - **SpringBoot 管理端**：在“人脸注册”流程中，被 SpringBoot 调用（SpringBoot 把图片发给 Flask，Flask 提取 128 维特征向量后返回给 SpringBoot）。

### 3. SpringBoot 管理端 (含 MySQL)

- **调用谁（主动请求）**：
  - **MySQL 数据库**：所有业务数据（用户、学生、告警、规则、视频源配置）的读写持久化。
  - **AI 算法**：在人脸注册流程中，主动调 Flask 提取特征；在 `/system/health` 探活时，调 Flask 检查 AI 服务状态。
  - **云服务器**：在查询视频源状态时，探活 Nginx 的流媒体端口是否在线。
- **被谁调用（提供给谁）**：
  - **前端**：向前端提供所有的业务 RESTful API（鉴权、CRUD、告警查询、统计等）。
  - **AI 算法**：被 Flask 调用，接收 Flask 推送过来的告警事件数据并入库。

### 4. 云服务器推拉流模块

- **调用谁（主动请求）**：
  - 无。Nginx 是被动的服务端，只接收请求，不主动向外发请求。
- **被谁调用（提供给谁）**：
  - **推流端（OBS/手机/摄像头）**：接收推流端的 RTMP 视频数据。
  - **AI 算法**：提供 RTMP 拉流地址，供 Flask 抽取视频帧。
  - **前端**：提供 9092 端口的 HTTP 服务，供前端直接访问和下载截图、录像文件。
  - **SpringBoot 管理端**：被 SpringBoot 探活端口连通性。
