# 生产环境配置指南

本文档说明三个子模块的生产环境配置方法：从 example 文件复制并填入真实值。

---

## 1. frontend

```bash
cp frontend/public/runtime-config.example.js frontend/public/runtime-config.js
```

编辑 `frontend/public/runtime-config.js`，填入真实地址：

```js
window.__SMART_CLASS_CONFIG__ = {
  API_BASE: "http://<后端服务器IP>:8080",
  AI_BASE: "http://<AI服务器IP>:5001",
  NGINX_BASE: "http://<Nginx服务器IP>:9092",
};
```

当前本地实际值：

```js
window.__SMART_CLASS_CONFIG__ = {
  API_BASE: "http://localhost:8080",
  AI_BASE: "http://localhost:5001",
  NGINX_BASE: "http://39.106.209.208:9092",
};
```

> **说明**：`API_BASE` 指向 Spring Boot 后端，`AI_BASE` 指向 Flask AI 服务，`NGINX_BASE` 指向 Nginx RTMP 服务。生产部署时将 `localhost` 替换为对应服务器的公网/内网 IP。

---

## 2. backend_ai

```bash
cp backend_ai/.env.example backend_ai/.env
```

编辑 `backend_ai/.env`，填入真实值：

```env
# Spring Boot 后端地址
SPRING_BASE_URL=http://<Spring Boot服务器IP>:8080

# AI 内部接口鉴权 Token（需与 backend_system 的 AI_INTERNAL_TOKEN 一致）
INTERNAL_TOKEN=<与backend_system一致的Token>
```

当前本地实际值：

```env
# Spring Boot 后端地址（同机部署，用 localhost）
SPRING_BASE_URL=http://localhost:8080

# AI 内部接口鉴权 Token（需与 backend_system 的 ai.internal-token 一致）
INTERNAL_TOKEN=smart-class-ai-internal-token-2026
```

> **说明**：`INTERNAL_TOKEN` 必须与 `backend_system` 的 `AI_INTERNAL_TOKEN` 保持一致，否则内部服务间调用会被拦截。

---

## 3. backend_system

```bash
cp backend_system/.env.example backend_system/.env
```

编辑 `backend_system/.env`，填入真实值：

```env
# ===== 数据库 =====
SPRING_DATASOURCE_URL=jdbc:mysql://<数据库IP>:3306/appdb?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true
SPRING_DATASOURCE_USERNAME=<数据库用户名>
SPRING_DATASOURCE_PASSWORD=<数据库密码>

# ===== JWT =====
AUTH_JWT_SECRET=<JWT密钥>

# ===== AI 服务 =====
AI_BASE_URL=http://<AI服务器IP>:5001
AI_INTERNAL_TOKEN=<与backend_ai一致的Token>

# ===== Nginx RTMP =====
NGINX_STAT_URL=http://<Nginx服务器IP>:9092/stat

# ===== CORS =====
CORS_ALLOWED_ORIGINS=http://<前端域名>*,http://<服务器IP>:*
```

当前本地实际值：

```env
# ===== 数据库 =====
SPRING_DATASOURCE_URL=jdbc:mysql://39.106.209.208:3306/appdb?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true
SPRING_DATASOURCE_USERNAME=appuser
SPRING_DATASOURCE_PASSWORD=MySQL@186strongpass3435

# ===== JWT =====
AUTH_JWT_SECRET=smart-class-monitor-system-jwt-secret-key-2026

# ===== AI 服务（同机部署，用 localhost）=====
AI_BASE_URL=http://localhost:5001
AI_INTERNAL_TOKEN=smart-class-ai-internal-token-2026

# ===== Nginx RTMP（同机部署，用 localhost）=====
NGINX_STAT_URL=http://39.106.209.208:9092/stat

# ===== CORS =====
CORS_ALLOWED_ORIGINS=http://localhost:*,http://127.0.0.1:*,http://39.106.209.208:*
```

> **说明**：
>
> - 数据库连接信息需根据实际 MySQL 实例填写
> - `AUTH_JWT_SECRET` 生产环境请使用高强度随机字符串
> - `AI_INTERNAL_TOKEN` 必须与 `backend_ai` 的 `INTERNAL_TOKEN` 一致
> - `CORS_ALLOWED_ORIGINS` 需包含前端访问的所有域名/IP

## 配置关联关系

```
┌─────────────┐         ┌──────────────┐         ┌──────────────┐
│  frontend   │────────▶│ backend_system│────────▶│  backend_ai  │
│             │         │              │         │              │
│ API_BASE ───┘         │ AI_BASE_URL ─┤         │ SPRING_BASE  │──▶ backend_system
│ AI_BASE ──────────────┼──────────────┤         │ INTERNAL_    │
│ NGINX_BASE ──▶ Nginx  │ AI_INTERNAL  │◀━━━━━━▶│   TOKEN      │
│                       │   TOKEN      │  一致   │              │
└─────────────┘         └──────────────┘         └──────────────┘
```

关键一致性约束：

- `backend_system` 的 `AI_INTERNAL_TOKEN` === `backend_ai` 的 `INTERNAL_TOKEN`
- `frontend` 的 `API_BASE` 指向 `backend_system` 的地址
- `frontend` 的 `AI_BASE` 指向 `backend_ai` 的地址
- `frontend` 的 `NGINX_BASE` 与 `backend_system` 的 `NGINX_STAT_URL` 指向同一 Nginx 服务
