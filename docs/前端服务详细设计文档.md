# 智慧教室前端服务接口与登录注册设计文档

版本：V0.4
日期：2026-07-08

## 1. 文档目的

本文档面向智慧教室前端开发，说明 Vue 前端的页面模块、登录入口、接口调用边界、字段约定、错误处理和后续扩展方式。

本版本严格对齐以下两份设计文档：

- `后端服务详细设计文档.md`
- `AI服务部分详细设计文档.md`

前端不得自行新增或假定后端未确认的正式接口。手机号注册当前只作为前端预留/申请注册界面，后端文档尚未确认 `/auth/register` 或 `/auth/register/code`，因此前端代码不调用这两个接口。

---

## 2. 三端服务边界

| 服务 | 默认端口 | 前端变量 | 前端调用范围 |
| --- | --- | --- | --- |
| SpringBoot 后端 | 8080 | `VITE_API_BASE` | 登录鉴权、视频源、人员、人脸注册、区域、规则、告警、统计、系统状态、操作日志 |
| Flask AI 服务 | 5000 | `VITE_AI_BASE` | 处理后视频流、AI 候选事件、AI 摘要、模型状态 |
| Nginx 静态服务 | 9092 | `VITE_NGINX_BASE` | 告警截图、自动录像、后续视频切片资源 |

前端调用原则：

1. 用户登录、鉴权、人员、区域、规则、告警等结构化数据全部走 SpringBoot。
2. 实时视频展示走 Flask `/video_feed/{stream_id}`。
3. 截图和录像由 SpringBoot 返回相对路径，前端拼接 `VITE_NGINX_BASE` 访问。
4. 前端不直接调用 AI 的人脸特征提取、配置刷新、告警入库接口。
5. 正式告警实时推送如需实现，应走 AI → SpringBoot 入库 → SpringBoot WebSocket/SSE 推送前端。

## 3. 登录与手机号注册入口设计

### 3.1 页面目标

前端打开后先判断本地是否存在有效登录状态：

- 未登录：展示登录/手机号注册预留界面。
- 已登录：展示智慧教室监控主界面。
- 后端认证接口不可用：允许演示登录兜底，避免前端白屏。

### 3.2 登录页面布局

登录入口采用左右分区：

| 区域 | 内容 |
| --- | --- |
| 左侧能力区 | 系统名称、JWT 鉴权、角色访问、接口一致性说明 |
| 右侧表单区 | 用户登录、手机号注册预留两个 Tab |

### 3.3 用户登录表单

| 字段 | 前端字段 | 后端字段 | 校验 |
| --- | --- | --- | --- |
| 用户名/工号 | `username` | `username` | 必填 |
| 密码 | `password` | `password` | 必填 |
| 记住登录状态 | `remember` | 不传后端 | 控制 Token 存储位置 |

提交后调用：

```http
POST /auth/login
Content-Type: application/json
```

请求体：

```json
{
  "username": "admin",
  "password": "123456"
}
```

成功响应按后端统一响应结构解析：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "jwt-token",
    "user_id": 1,
    "username": "admin",
    "role": "admin",
    "nickname": "管理员",
    "avatar_url": ""
  }
}
```

前端处理：

1. 保存 `token`。
2. 保存用户信息。
3. 后续 SpringBoot 请求自动添加 `Authorization: Bearer <token>`。
4. 进入监控主界面并加载 `/streams`、`/alerts`、`/rules` 等数据。

### 3.4 当前用户信息

登录态恢复或刷新页面后调用：

```http
GET /auth/info
Authorization: Bearer <token>
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "user_id": 1,
    "username": "admin",
    "role": "admin",
    "nickname": "管理员",
    "avatar_url": ""
  }
}
```

如果返回 `401`，前端清除 Token 并回到登录页。

### 3.5 退出登录

后端文档中 `/auth/logout` 状态为“待定”。前端保留调用，但不能依赖该接口一定存在：

```http
POST /auth/logout
```

前端策略：

- 如果接口存在：调用成功后清除本地 Token。
- 如果接口不存在或失败：仍清除本地 Token，回到登录页。

### 3.6 手机号注册预留流程

后端详细设计文档当前没有确认以下接口：

- `POST /auth/register`
- `POST /auth/register/code`

因此前端当前实现为“手机号注册预留/申请注册界面”，不向后端提交注册请求。

表单字段：

| 字段 | 前端字段 | 说明 |
| --- | --- | --- |
| 手机号 | `phone` | 11 位手机号，本地校验 |
| 姓名 | `name` | 申请人姓名 |
| 申请角色 | `role` | `teacher` 或 `admin` |
| 密码 | `password` | 本地填写，不提交后端 |
| 确认密码 | `confirmPassword` | 与密码一致 |
| 申请说明 | `remark` | 任课班级、管理范围或申请原因 |

提交后前端提示：手机号注册接口尚未在后端文档中确认，请联系管理员创建账号。

后续如团队确认注册接口，必须先补 OpenSpec change，再同步后端 DTO、Swagger 和本文档。

| 接口 | 方法 | 前端用途 | 入参 | 返回 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `/video_feed/{stream_id}` | GET | 实时处理后视频 | `stream_id` | MJPEG 流 | 前端 `<img>` 直接渲染 |
| `/analysis/events` | GET | AI 候选事件 | `stream_id`、`event_type`、`time_range` | 候选事件列表 | 不替代正式告警 |

### 4.3 AI 内部接口，前端不得直接调用

| 接口 | 方法 | 调用方 | 用途 |
| --- | --- | --- | --- |
| `/face/feature/extract` | POST | SpringBoot | 人脸注册时提取 512 维特征 |

| `POST /alerts/ai` | POST | AI → SpringBoot | AI 确认告警后入库，前端不得调用 |
| `GET /students/face-features` | GET | AI → SpringBoot | AI 同步特征库，前端不得调用 |

## 5. 字段与响应约定

### 5.1 统一响应

普通响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

分页响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "records": [],
    "page": 1,
    "page_size": 10,
    "total": 0
  }
}
```

前端兼容 `records`、`list`、`rows` 三种列表字段，但正式接口建议统一为 `records`。

### 5.2 命名

- 后端响应字段使用 `snake_case`。
- 前端展示层可直接使用 `snake_case` 字段，避免额外映射混乱。
- 文件资源字段：AI/SpringBoot 入库用 `snapshot_path`、`record_path`；SpringBoot 返回前端可用 `snapshot_url`、`record_url`。

### 5.3 Token

- 登录成功后保存 JWT。
- 请求 SpringBoot 业务接口时添加：`Authorization: Bearer <token>`。
- Flask AI 展示接口初版不要求前端 JWT，但生产环境可通过网关统一保护。
- AI 内部接口使用单独内部 Token 或签名，不复用前端 JWT。

### 5.4 错误码处理

| code | 含义 | 前端处理 |
| --- | --- | --- |
| 400 | 参数错误 | 表单提示具体字段 |
| 401 | 未登录或 Token 失效 | 清 Token，返回登录页 |
| 403 | 无权限 | 提示角色权限不足 |
| 404 | 资源不存在 | 显示空态或错误提示 |
| 409 | 重复或状态冲突 | 显示冲突原因 |
| 500 | 服务异常 | 显示降级数据或重试按钮 |

## 6. 页面模块设计

### 6.1 认证入口

- 位置：应用首屏，未登录时展示。
- 能力：用户登录、手机号注册预留、演示登录兜底、接口契约提示。
- 登录成功后进入主监控页面。

### 6.2 实时监控页

- 视频来源：`VITE_AI_BASE + /video_feed/{stream_id}`。
- 告警来源：SpringBoot `/alerts` 轮询，后续可接 SpringBoot WebSocket/SSE。
- AI 候选事件：`/analysis/events`。
- AI 摘要：`/analysis/summary/{stream_id}`，初版可 mock。

### 6.3 告警管理页

- 列表：`GET /alerts`。
- 详情：`GET /alerts/{id}`。
- 状态处理：`PUT /alerts/{id}/status`。
- 截图/录像：`VITE_NGINX_BASE + snapshot_url/record_url`。

### 6.4 区域规则页

- 区域：`GET /zones`，后续 POST/PUT/DELETE 按后端待定接口扩展。
- 规则：`GET /rules`、`PUT /rules/{id}`。
- 规则字段必须支持 `threshold_seconds`、`confidence_threshold`、`cooldown_seconds`、`config_json`。
- 区域字段必须预留 `safe_distance`，用于 `danger_zone_approach`。

### 6.5 人员与人脸库页

- 人员列表：`GET /students`。
- 新增人员：`POST /students`。
- 人脸注册：`POST /students/{id}/face`，前端上传 base64 图片。
- 前端不展示完整 `feature_vector`。

### 6.6 系统状态页

- 系统健康：`GET /system/health`。
- AI 状态：`GET /model/status`。
- 视频源状态：`GET /streams/{stream_id}/status`。

## 7. 告警类型和规则枚举

当前以后端详细设计文档 V0.3 为准：

| 类型 | 中文含义 | 本期状态 |
| --- | --- | --- |
| `face_recognized` | 人脸识别成功 | 本期 |
| `stranger_detected` | 陌生人检测 | 本期 |
| `danger_zone_intrusion` | 危险区域进入 | 本期 |
| `danger_zone_stay` | 危险区域停留超时 | 本期 |
| `danger_zone_approach` | 危险区域接近预警 | 本期 |
| `phone_usage` | 使用手机 | 本期 |
| `head_down` | 长时间低头 | 本期 |
| `leave_seat` | 长时间离座 | 本期 |
| `flame_detected` | 明火检测 | 本期 |
| `fall_detected` | 摔倒检测 | 本期 |
| `stream_offline` | 视频源离线 | 本期 |
| `spoofing_attack` | 防欺骗攻击 | 后续扩展 |
| `acoustic_anomaly` | 异常声学事件 | 后续扩展 |

正式告警状态：

```text
unhandled
processing
handled
false_alarm
ignored
```

AI 候选事件状态不进入正式告警状态机，可在前端展示为 `candidate`、`confirmed`、`expired`。

## 8. 资源路径设计

| 字段 | 来源 | 前端处理 |
| --- | --- | --- |
| `snapshot_url` | SpringBoot 告警详情 | 如果是相对路径，拼接 `VITE_NGINX_BASE` |
| `record_url` | SpringBoot 告警详情 | 指向 Nginx 自动录像文件 |
| `video_clip_url` | 后续扩展 | 初版可为空，不虚构 `/clips/` 地址 |

前端不得把服务器 IP 写死到组件里。

## 9. 降级与演示策略

| 场景 | 前端策略 |
| --- | --- |
| `/auth/login` 不可用 | 允许演示登录，显示演示提示 |
| `/auth/info` 失败 | 保留本地用户或回到登录页，视 Token 是否存在 |
| 业务接口失败 | 使用 mock 数据兜底，避免白屏 |
| 视频流失败 | 展示课堂示意图和视频源异常状态 |
| AI 摘要失败 | 使用本地默认研判文案 |
| Nginx 资源为空 | 显示暂无证据或资源不可用 |

## 10. 与后端/AI 文档一致性结论

| 事项 | 前端采用方案 |
| --- | --- |
| 登录接口 | 使用后端已确定的 `POST /auth/login` |
| 当前用户 | 使用后端已确定的 `GET /auth/info` |
| 退出登录 | 保留 `/auth/logout`，但按待定接口降级处理 |
| 手机号注册 | 前端预留，不声明正式接口 |
| 人脸注册 | 前端 → SpringBoot `/students/{id}/face` → AI `/face/feature/extract` |
| AI 模型状态 | 使用 `/model/status`，不使用 `/ai/model/status` |
| 告警入库 | AI 调 SpringBoot `POST /alerts/ai`，前端不调用 |
| 告警推送 | SpringBoot 入库后推送或前端轮询 `/alerts` |
| 截图录像 | SpringBoot 返回相对路径，前端拼接 `VITE_NGINX_BASE` |
| 告警详情录像字段 | 使用 `record_url`，不依赖 `video_clip_url` |
