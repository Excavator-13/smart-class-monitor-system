# 生产环境配置指南

本文档列出所有生产环境所需的配置文件及其真实值，组员按路径复制即可完成部署。

---

## 1. `backend_ai/config/model.yaml`

控制 AI 模型的启用/禁用和参数。`enabled` 字段可按需调整，`device: auto` 会自动选择 CUDA 或 CPU。

```yaml
device: auto
models:
  face:
    enabled: false
    provider: insightface
    name: buffalo_l
    providers:
      - CUDAExecutionProvider
    ctx_id: 0
    det_size: [640, 640]
    feature_dim: 512
    similarity_threshold: 0.45
  behavior:
    enabled: true
    provider: ultralytics
    name: yolov8n-coco
    weights: models/yolo/yolov8n.pt
    confidence_threshold: 0.6
  zone:
    enabled: true
    provider: rule
    safe_distance: 0.05
  fire:
    enabled: false
    provider: ultralytics
    weights: models/yolo/yolo_fire.pt
    confidence_threshold: 0.25
    max_detections: 20
    min_bbox_area: 1000
    cooldown_seconds: 10
  anti_spoof:
    enabled: false
    provider: rule
    blink_threshold_seconds: 5.0
    texture_variance_threshold: 8.0
    deepfake_weights: models/mesonet/meso4_weights.pth
  audio:
    enabled: false
    provider: signal
    sample_rate: 16000
    window_ms: 1000
```

## 4. `frontend/public/runtime-config.js`

此文件在 `public/` 目录下，**不经过 Webpack 构建**，部署后可直接修改无需重新打包。

```js
window.__SMART_CLASS_CONFIG__ = {
  API_BASE: "http://localhost:8080",
  AI_BASE: "http://localhost:5001",
  NGINX_BASE: "http://39.106.209.208:9092",
};
```

| 字段         | 指向                                |
| ------------ | ----------------------------------- |
| `API_BASE`   | Spring Boot 后端地址                |
| `AI_BASE`    | AI 服务地址                         |
| `NGINX_BASE` | Nginx RTMP 流媒体地址（含截图服务） |

> 如果前端部署在服务器上，`API_BASE` 和 `AI_BASE` 可能需要改为服务器实际 IP 或域名。
