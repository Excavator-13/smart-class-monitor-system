# 智慧教室监测系统 — 前后端联调指南

## 一、环境准备

### 1.1 服务启动顺序

```
① MySQL 数据库（确保 appdb 库存在，schema.sql 已执行）
② SpringBoot 后端  →  http://localhost:8080
③ Flask AI 服务     →  http://localhost:5001
④ Nginx RTMP 流媒体 →  http://39.106.209.208:9092
⑤ Vue 前端          →  http://localhost:5173
```

### 1.2 测试数据说明

SpringBoot 每次启动会自动清空表并插入测试数据（由 `DataInitializer` 完成），无需手动操作。

| 表            | 数据量 | 关键内容                                                                           |
| ------------- | ------ | ---------------------------------------------------------------------------------- |
| user          | 2 条   | admin / admin123（管理员）、teacher / teacher123（教师）                           |
| video_stream  | 2 条   | classroom_01（RTMP: 39.106.209.208:9090）、classroom_02（备用）                    |
| student       | 5 条   | 张三/李四/王五（已注册人脸）、赵六/孙七（未注册）                                  |
| behavior_rule | 6 条   | 手机违规、明火、摔倒、低头、陌生人、区域入侵                                       |
| danger_zone   | 2 条   | 讲台危险区域（danger）、手机禁用区（phone_forbidden），均属 classroom_01           |
| alert_event   | 4 条   | 低头(unhandled)、区域入侵(processing)、陌生人(high/unhandled)、已处理低头(handled) |

### 1.3 前端环境变量（frontend/.env）

```
VITE_API_BASE=http://localhost:8080     # SpringBoot
VITE_AI_BASE=http://localhost:5001      # Flask AI
VITE_NGINX_BASE=http://39.106.209.208:9092  # Nginx 静态资源
```

### T2 Dashboard 初始化加载

- ✅ 9 个请求是否全部发出

- ✅ 页面各区域数据正确渲染（视频源下拉有 2 个选项、告警列表有 4 条等）

### T8 风险评分（结合测试告警数据）

前端风险评分引擎根据告警类型、等级、置信度动态计算。当前 4 条测试告警的评分预期：

| 告警                       | classifyRiskType | 基础分 | 等级加成 | 置信度加成 | 最终分 |
| -------------------------- | ---------------- | ------ | -------- | ---------- | ------ |
| 陌生人出现 (high, 0.88)    | stranger         | 70     | +5       | +3         | 78     |
| 区域入侵 (warning, 0.76)   | zone             | 58     | +2       | +0         | 60     |
| 长时间低头 (warning, 0.82) | behavior         | 42     | +2       | +2         | 46     |
| 已处理低头 (info, 0.65)    | behavior         | 42     | -8       | -2         | 32     |

> 注意：风险评分由前端 `activeRiskEvents` 计算属性驱动，它合并了 `alerts`（SpringBoot）和 `analysisEvents`（Flask AI）两个来源。当前测试数据只有 SpringBoot 告警，如果 Flask AI 也返回了事件，评分会叠加。

## 四、常见问题排查

| 现象                           | 可能原因                                    | 排查方法                                                               |
| ------------------------------ | ------------------------------------------- | ---------------------------------------------------------------------- |
| 登录提示"用户名或密码错误"     | DataInitializer 未执行或数据库连接异常      | 检查 SpringBoot 启动日志中是否有 `DataInitializer: 测试数据初始化完成` |
| 页面空白无数据                 | 后端接口返回格式与前端 normalize 不匹配     | DevTools Network 查看响应体，确认 `code: 0` 且 `data` 结构正确         |
| 视频画面不显示                 | Flask 未启动或 RTMP 无推流                  | 浏览器直接访问 `http://localhost:5001/video_feed/classroom_01`         |
| 告警列表为空                   | `/alerts` 返回的分页结构不含 `records` 字段 | 检查后端返回是否为 `{code:0, data:{records:[...], total:N}}`           |
| 风险评分为 0                   | 告警数据中缺少 `target_info` 或 `bbox`      | 检查告警的 `target_info` JSON 是否包含 `bbox` 数组                     |
| AI 标注框不显示                | `analysisEvents` 为空（Flask 未返回事件）   | 这是正常的——AI 标注框来自 Flask 事件，不是 SpringBoot 告警             |
| 系统状态页面 AI/RTMP 显示 down | Flask 或 Nginx 未启动                       | 确认对应服务是否运行                                                   |
| CORS 报错                      | 后端 CorsConfig 未覆盖前端地址              | 检查 `application.yml` 中 `cors.allowed-origins` 配置                  |
