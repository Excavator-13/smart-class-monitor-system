# 接口设计

## 1. 接口概述

系统采用前后端分离架构，包含两个后端服务：
- **Spring Boot 后端**（端口 8080）：业务数据管理、用户认证、告警存储
- **Flask AI 后端**（端口 5000）：视频流分析、AI 推理、事件检测

前端通过 HTTP RESTful API 与后端交互，后端服务之间通过内部接口通信。

---

## 2. 通用规范

### 2.1 请求格式

- 内容类型：`application/json`
- 字符编码：UTF-8
- 认证方式：JWT Token（前端请求）/ X-Internal-Token（内部服务调用）

### 2.2 响应格式

**标准响应结构**：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

**分页响应结构**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [...],
    "page": 1,
    "page_size": 10,
    "total": 100
  }
}
```

**错误响应结构**：

```json
{
  "code": 40001,
  "message": "invalid image",
  "data": null
}
```

### 2.3 错误码定义

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 40001 | 无效图像 |
| 40002 | 未检测到人脸 |
| 40003 | 检测到多张人脸 |
| 40401 | 视频流不存在 |

## 4. Flask AI 后端接口

### 4.1 视频流接口

#### GET /video_feed/{stream_id}

**说明**：返回 MJPEG 实时流

**路径参数**：`stream_id` - 视频源 ID

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|

**响应**：`multipart/x-mixed-replace; boundary=frame`

每帧格式：

```
--frame
Content-Type: image/jpeg

<JPEG 二进制数据>
```

## 10. 新增事件类型（v0.4.0）

AI 服务新增以下事件类型，通过 POST /internal/ai/alerts 推送至 Spring Boot：

| 事件类型 | 中文名称 | 默认等级 | 说明 |
|----------|----------|----------|------|
| spoof_detected | 活体检测异常 | high | 眨眼检测/纹理分析判定为照片或视频攻击 |
| deepfake_detected | AI换脸攻击 | high | MesoNet CNN 判定为换脸伪造 |
| abnormal_sound | 异常声学事件 | warning/high | 尖叫、玻璃破碎、爆炸等异常声音 |
| fall_detected | 人员摔倒 | high | 人体宽高比异常（水平状态） |
| crowd_gathering | 异常聚集 | warning | 多人中心点距离过近 |
| stream_offline | 视频流中断 | warning | 超过 10 秒无新帧 |
