# 钉钉告警通知系统说明书

## 一、概述

本系统集成钉钉企业内部应用机器人，实现告警事件的实时推送与逐级上报。当 AI 检测到异常事件时，自动向指定钉钉群发送告警消息并 @相关负责人；若在设定时间内未收到"已处理"回复，则自动上报至直属上级，直至全员通知。

- **群名称**：项目告警群
- **机器人名称**：智慧教室告警

---

## 二、功能说明

### 2.1 告警通知

AI 服务检测到异常事件后，自动向钉钉群发送告警消息，包含告警类型、发生位置、置信度等关键信息，并 @第一责任人。

### 2.2 逐级上报

告警发出后进入上报流程：

| 步骤 | 时间 | 接收人 | 内容 |
|---|---|---|---|
| 1 | 即时 | 项重善（第一责任人） | 告警通知 |
| 2 | 30 秒后 | 章志超（直属上级） | 告警升级 |
| 3 | 60 秒后 | @ 全体成员 | 紧急升级 |

### 2.3 响应处理

责任人在群内回复 **"已处理"** 或 **"误报"**，系统自动停止上报链条，无需手动操作。

### 2.4 组织架构自动查询

系统通过钉钉通讯录 API 自动获取责任人的直属上级，无需手动配置上报关系。当组织架构变更时，上报链同步更新。

## 三、技术实现

### 3.1 架构

```
AI 事件 → alert_client.py → 发钉钉消息 (Webhook)
                           ↓
                      dingtalk_service.py
                           ↓
            Stream 监听群消息 → 检测"已处理" → 停止上报
                           ↓
            定时任务 → 30 秒检查 → 未响应 → 查 API 上级 → 升级通知
```

### 3.2 核心模块

| 模块 | 文件 | 说明 |
|---|---|---|
| 告警推送 | `backend_ai/services/dingtalk_service.py` | 发送消息、逐级上报、Stream 监听 |
| 告警集成 | `backend_ai/services/alert_client.py` | 告警推送时自动调用钉钉通知 |
| 应用配置 | `backend_ai/config/app.yaml` | 责任人映射、上报超时等参数 |

### 3.3 依赖服务

| 服务 | 说明 |
|---|---|
| 钉钉开放平台 | 企业内部应用，AppKey/AppSecret 鉴权 |
| 通讯录 API | 自动获取用户信息与直属上级关系 |
| Stream 模式 | WebSocket 长连接，实时接收群消息 |
| Webhook | HTTP 推送，支持 @手机号蓝色通知 |

## 四、配置说明

### 4.1 责任人配置

配置路径：`backend_ai/config/app.yaml` → `dingtalk.persons`

```yaml
persons:
  项重善:
    name: "项重善"
    mobile: "18601033435"
  章志超:
    name: "章志超"
    mobile: "15270985055"
```

### 4.2 上报参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `step_timeout` | 30 秒 | 每级上报等待时间 |
| `primary` | "项重善" | 第一责任人 |

### 4.3 响应关键词

群内回复以下任意词即停止上报：

- `已处理`
- `误报`

## 六、故障处理

| 现象 | 可能原因 | 处理方式 |
|---|---|---|
| 未收到告警消息 | Token 过期或网络异常 | 检查 app.yaml 中 AppKey/AppSecret 是否正确 |
| @人不显示蓝色 | Webhook token 失效 | 在钉钉开放平台重新获取 Webhook 地址 |
| 上报链未自动停止 | Stream 未启动 | 确认 start_stream() 在 app.py 中已调用 |
| 查不到直属上级 | 通讯录权限未开启 | 钉钉开放平台 → 权限管理 → 开启相关权限 |
