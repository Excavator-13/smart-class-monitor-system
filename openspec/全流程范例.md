# OpenSpec 全流程范例

以「加法计算」功能为例，展示 `/opsx:explore → propose → apply → archive` 四步。

---

## Step 1: /opsx:explore — 先讨论思路

```
You: /opsx:explore

AI: What would you like to explore?

You: 我需要一个加法功能，接收两个数字返回和。
      不确定放在哪个文件，也不确定用什么参数格式。

AI: 让我看一下你的项目结构...
    
    backend_ai/ 下加一个 adder.py 最合理。
    建议用 Flask GET 接口，参数放 URL 上：
    GET /api/add?a=1&b=2 → {"result": 3}

    简单清晰，不需要数据库。需要我开始写 spec 吗？

You: 好，就这么做。
```

---

## Step 2: /opsx:propose — 生成全套文档

```
You: /opsx:propose add-api

AI: Created openspec/changes/add-api/
    ✓ proposal.md   — 动机与范围
    ✓ specs/        — 功能规范
    ✓ design.md     — 技术方案
    ✓ tasks.md      — 实现清单
    Ready for implementation!
```

### 生成的 proposal.md
```markdown
# 加法 API

## 动机
需要提供一个后端加法接口，供前端调用。

## 范围
- 接收两个数字参数，返回和
- 校验非法输入
- 不做数据库存储
```

### 生成的 specs/add-api.md
```markdown
## Requirement: 加法计算
系统必须接收 a、b 两个数字参数，返回它们的和。
系统不得接受非数字输入。

### Scenario: 1+1=2
- GIVEN 用户输入 a=1, b=1
- WHEN 系统计算
- THEN 返回 {"result": 2}

### Scenario: 非法输入
- GIVEN 用户输入 a="abc", b=2
- WHEN 系统校验
- THEN 返回 400 错误 "请输入数字"
```

### 生成的 design.md
```markdown
## 技术方案
- 框架：Flask
- 接口：GET /api/add
- 参数：a (float), b (float)
- 返回：JSON
```

### 生成的 tasks.md
```markdown
- [ ] 在 backend_ai/ 下创建 adder.py
- [ ] 实现 GET /api/add 接口
- [ ] 添加参数校验
- [ ] 测试 1+1=2
```

---

## Step 3: /opsx:apply — AI 写代码

```
You: /opsx:apply

AI: Implementing tasks...
    ✓ 1.1 Create backend_ai/adder.py
    ✓ 1.2 Implement add() function
    ✓ 1.3 Add input validation
    All tasks complete!
```

### 生成的代码（backend_ai/adder.py）
```python
# 加法 API
# 对应 openspec/changes/add-api/specs/add-api.md

from flask import Flask, request, jsonify

app = Flask(__name__)


@app.route('/api/add', methods=['GET'])
def add():
    """接收 a、b 两个数字参数，返回和"""
    a = request.args.get('a', type=float)
    b = request.args.get('b', type=float)

    # Scenario: 非法输入（spec 要求的）
    if a is None or b is None:
        return jsonify({'error': '请输入数字'}), 400

    # Scenario: 1+1=2（spec 要求的）
    return jsonify({'result': a + b})


if __name__ == '__main__':
    app.run(debug=True)
```
