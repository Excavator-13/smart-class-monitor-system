## Requirement: 钉钉告警通知与逐级上报
系统必须在检测到安全事件时通过钉钉群机器人发送告警通知，支持逐级上报和回复停止。
系统必须支持长按回复精确停止单个事件，而非全部停止。
系统不得将已识别人员、低头等非安全事件推送到钉钉。

### Scenario: 发送首条告警通知
- GIVEN AI 检测到一个安全事件（陌生人入侵/区域入侵/明火/摔倒/聚集/声音异常/视频中断）
- WHEN 系统调用 trigger_alert 发送通知
- THEN 钉钉群收到一条 text 消息（蓝色 @责任人）
- AND 消息底部包含唯一的 事件ID
- AND 如果有截图，紧随其后发送一条 markdown 消息展示截图

### Scenario: 逐级上报
- GIVEN 责任人 45 秒内未回复「已处理」
- WHEN 系统自动升级到上级
- THEN 发送 text 消息 @上级，包含同一事件ID
- AND 不重复发送截图

### Scenario: 最终@全体成员
- GIVEN 逐级上报完成后仍无人响应
- WHEN 系统执行最后一级上报
- THEN 发送 text 消息 @全体成员

### Scenario: 长按回复精确停止
- GIVEN 群内有用户长按告警消息并回复「已处理」
- WHEN 系统收到回复，解析到被回复的原文中的事件ID
- THEN 只停止该事件ID对应的告警的上报链
- AND 机器人回复「已标记事件 evt_xxx ，停止上报」

### Scenario: 普通回复停止全部
- GIVEN 用户在群内直接发送「已处理」（未回复任何消息）
- WHEN 系统无法匹配到特定事件
- THEN 停止所有活动告警的上报链
- AND 机器人回复「未找到对应事件ID，已停止全部上报链」

### Scenario: 联系人变更
- GIVEN 前端告警管理页修改了负责人或通知人
- WHEN 系统保存设置
- THEN 下次告警使用新的责任人链
- AND 逐级上报顺序同步更新
