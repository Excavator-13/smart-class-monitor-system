## Why
现有钉钉通知无法区分多个并发事件，一个「已处理」停掉所有告警。逐级上报消息使用 Markdown 无法蓝色 @ 人。

## What Changes
1. 钉钉通知发送改为 text 消息实现蓝色 @
2. 引入事件ID机制，每条消息带唯一 ID
3. 长按回复自动匹配事件ID，只停对应告警
4. 逐级上报仅发 text + @，不再重复发截图
5. 上报间隔改为 45 秒

## Capabilities
- 同一内容的两条告警不冲突，各有独立事件ID
- 回复「已处理」精确停止单个事件
- 无事件ID的回复走全部停止（兼容旧行为）
- 机器人回复提示明确告知停止的是哪个事件
