# 登录注册入口界面

## 动机

当前前端原型打开后直接进入监控主界面，缺少系统入口、手机号注册和用户登录流程。为了匹配智慧教室系统的角色访问、后续鉴权和接口文档，需要增加一个清晰、可演示、可联调的认证入口。

## 范围

- 增加登录/注册切换式入口界面。
- 支持用户登录表单，保留 `/auth/login` 和 `/auth/info` 接口调用。
- 支持手机号注册/申请注册表单，但不声明未被后端文档确认的注册接口为正式接口。
- 认证接口严格以后端服务详细设计文档为准：已确定 `/auth/login`、`/auth/info`，`/auth/logout` 为待定。
- 登录成功后进入现有智慧教室监控主界面。
- 后端暂不可用时使用 mock 兜底，避免页面白屏。
- 扩充 `前端服务接口文档.md` 为包含认证模块的详细前端设计文档。

## 不做

- 不实现真实短信服务。
- 不修改 SpringBoot 后端代码。
- 不新增或假定后端未确认的 `/auth/register`、`/auth/register/code` 接口。
- 不新增数据库表。
- 不引入完整路由系统。
- 不改变现有监控主界面的核心布局和交互。

## 影响模块

- frontend
- docs
- openspec
