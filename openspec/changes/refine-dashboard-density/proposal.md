# 优化前端页面信息密度

## 动机
当前实时监控页的 AI 研判助手视觉权重偏弱，其他四个页面存在较多空白区域。需要在不改变后端接口契约的前提下，提高重点模块辨识度，并让告警、规则、人脸库、系统状态页面更像可交付的后台界面。

## 范围
- 放大并强化实时监控页中的 AI 研判助手。
- 为告警管理页增加统计卡片和处置辅助信息。
- 为区域规则页增加规则统计、规则模板和下发状态信息。
- 为人员与人脸库页增加人员统计、人脸注册进度和陌生人核验信息。
- 为系统状态页增加运行统计、链路概览和操作日志信息。

## 不做
- 不新增真实后端接口。
- 不修改数据库结构。
- 不接入真实图表库。
- 不改变现有导航结构。

## 影响模块
- frontend
- openspec
