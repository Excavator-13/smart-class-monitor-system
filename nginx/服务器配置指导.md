# 异常事件录像回放 — 服务器配置指导

## 概述

本功能通过 nginx-rtmp `exec_publish` 钩子启动 FFmpeg 实时切片，每 30 秒产出一个 flv 片段，自动转 mp4 入库。AI 端根据事件时间算出对应片段路径，前端弹出播放器回放。

## 1. 部署脚本

将以下脚本复制到服务器 `/opt/scripts/` 目录：

```bash
# 在本地项目 nginx/ 目录下
scp start_segment.sh root@<SERVER_IP>:/opt/scripts/
scp stop_segment.sh root@<SERVER_IP>:/opt/scripts/
scp flv2mp4.sh root@<SERVER_IP>:/opt/scripts/
scp clean_records.sh root@<SERVER_IP>:/opt/scripts/
```

赋予执行权限：

```bash
ssh root@<SERVER_IP> "chmod +x /opt/scripts/start_segment.sh /opt/scripts/stop_segment.sh /opt/scripts/flv2mp4.sh /opt/scripts/clean_records.sh"
```

## 2. 创建切片存储目录

确认 FFmpeg 套件同时提供 `ffprobe`（转码入库脚本使用它读取 MP4 真实时长，避免关键帧导致的切片时间空洞）：

```bash
ffmpeg -version
ffprobe -version
```

```bash
ssh root@<SERVER_IP> "mkdir -p /usr/local/rtmp_video/segments"
```

## 3. 更新 nginx 配置

将项目 `nginx/nginx.conf` 的内容更新到服务器 nginx 配置文件中（通常在 `/usr/local/nginx/conf/nginx.conf`）。

关键改动：

- rtmp application live 新增 `exec_publish` 和 `exec_publish_done` 指令
- http 9092 server 新增 `location /records/` 静态服务

验证配置语法：

```bash
ssh root@<SERVER_IP> "/usr/local/nginx/sbin/nginx -t"
```

重载 nginx：

```bash
ssh root@<SERVER_IP> "/usr/local/nginx/sbin/nginx -s reload"
```

## 4. 重启 flv2mp4.sh

如果 `flv2mp4.sh` 已注册为 systemd 服务（推荐），更新脚本后重启服务即可：

```bash
ssh root@<SERVER_IP> "systemctl restart flv2mp4"
```

如果未使用 systemd，手动重启：

```bash
# 查找并停止旧进程
ssh root@<SERVER_IP> "pkill -f flv2mp4.sh"

# 启动新版本（后台运行）
ssh root@<SERVER_IP> "nohup /opt/scripts/flv2mp4.sh > /var/log/flv2mp4.log 2>&1 &"
```

## 5. 配置定时清理

编辑 crontab：

```bash
ssh root@<SERVER_IP> "crontab -e"
```

添加（每天凌晨 3 点清理过期文件）：

```
0 3 * * * /opt/scripts/clean_records.sh >> /var/log/clean_records.log 2>&1
```

## 6. FFmpeg 僵尸进程检查（可选但推荐）

添加 cron 每小时检查一次：

```
0 * * * * find /tmp -name 'ffmpeg_seg_*.pid' -exec sh -c 'pid=$(cat "$1"); [ -n "$pid" ] && ! kill -0 "$pid" 2>/dev/null && rm -f "$1"' _ {} \;
```

## 7. 验证

### 7.1 验证切片产出

1. 用 OBS 或 ffmpeg 推一路测试流到 `rtmp://<SERVER_IP>:9090/live/test_stream`
2. 等待 30 秒以上
3. 检查切片目录：

```bash
ssh root@<SERVER_IP> "ls -la /usr/local/rtmp_video/segments/$(date +%Y%m%d)/"
```

应该看到类似 `test_stream-2026-07-13-XX_XX_XX.flv` 的文件。

### 7.2 验证自动转码入库

等待 flv2mp4.sh 处理后，检查：

```bash
ssh root@<SERVER_IP> "ls -la /usr/local/rtmp_video/segments/$(date +%Y%m%d)/*.mp4"
```

检查数据库：

```bash
ssh root@<SERVER_IP> "mysql -uroot -p'MySQL@186zhunshixiaban3435' appdb -e \"SELECT * FROM recording_file WHERE source_type='segment' ORDER BY id DESC LIMIT 5;\""
```

`duration_seconds` 应接近实际媒体时长但不要求严格等于 30；`ended_at` 应根据真实时长覆盖到文件最后一秒。

### 7.3 验证 nginx 静态服务

在浏览器中访问：

```
http://<SERVER_IP>:9092/records/<YYYYMMDD>/<stream_id>-<YYYY-MM-DD-HH_MM_SS>.mp4
```

应能播放或下载 mp4 文件。

### 7.4 验证推流断开时 FFmpeg 进程清理

1. 停止推流
2. 检查 PID 文件是否已删除：

```bash
ssh root@<SERVER_IP> "ls /tmp/ffmpeg_seg_*.pid 2>/dev/null || echo 'All PID files cleaned'"
```

3. 检查 FFmpeg 进程是否已终止：

```bash
ssh root@<SERVER_IP> "pgrep -f 'ffmpeg.*segment' || echo 'No segment FFmpeg processes'"
```

## 8. AI 端配置

AI 端配置已写入 `config/app.yaml` 的 `recording` 节。如需覆盖，可设置环境变量：

```bash
export RECORDING_SEGMENT_SECONDS=30
export RECORDING_SEGMENT_DIR=/records
```

重启 AI 服务即可生效。

## 9. 前端

前端无需额外配置。告警列表中 `record_url` 有值时"录像"按钮自动启用，点击弹出播放器并 seek 到事件发生位置。

## 注意事项

- **推流端 GOP 设置**：建议设为 2~4 秒，确保切片边界精确。OBS 中在"输出 > 高级 > 关键帧间隔"设置。
- **磁盘空间**：每路流 1Mbps 码率约 10GB/天，注意监控磁盘使用率。
- **`-c copy` 模式**：不重编码，CPU 开销极低，但要求源流关键帧间隔合理。
- **时区**：AI 端和服务器必须使用同一时区（默认 UTC+8），否则路径计算会偏差。
